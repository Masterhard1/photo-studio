# Фотостудия «Портретная» — сайт-визитка

Простой сайт на чистом Node.js (без фреймворков) с админкой для контента.

## Запуск (первый раз)

1. Скопируй `.env.example` в `.env`:
   ```
   copy .env.example .env
   ```
2. Открой `.env` и задай `ADMIN_PASSWORD_PRIMARY` (пароль мамы) и `ADMIN_PASSWORD_BACKUP` (резервный вход) — каждый не короче 10 символов, не похож на `1234567890` или номер телефона студии. Без этого сервер откажется запускаться и явно скажет, что не так.
3. Запусти сервер:
   ```
   node index.js
   ```
4. Открой `http://localhost:3000` — сайт, `http://localhost:3000/admin` — админка.

После первого запуска пароли сохраняются в виде хэшей в `data/admin-config.json` / `data/admin-config-backup.json` (эти файлы не попадают в git). `.env` дальше не используется для входа — он нужен только для самой первой инициализации. Сменить пароль можно прямо в админке.

## Деплой с HTTPS

Если на сервере есть домен и настоящий TLS-сертификат (например, через Let's Encrypt/certbot), укажи в `.env`:

```
SSL_KEY_PATH=/путь/к/privkey.pem
SSL_CERT_PATH=/путь/к/fullchain.pem
HTTPS_PORT=443
```

Тогда сервер сам поднимет HTTPS на `HTTPS_PORT`, а обычный HTTP (`PORT`) будет только редиректить на https — отдельный реверс-прокси не нужен. Если сертификатов нет — сервер работает по обычному HTTP (так и должно быть при локальной разработке).

Если вместо этого TLS терминируется на стороне хостинга/реверс-прокси (например, через nginx), `.env`-переменные SSL_* не нужны — просто убедись, что прокси передаёт заголовок `X-Forwarded-Proto: https`, тогда сессионная cookie всё равно будет помечена `Secure`.
